# Security Policy

## セキュリティポリシー

### サポート対象バージョン

現在メンテナンスされているバージョン：

| バージョン | サポート状況  |
| ---------- | ------------- |
| latest     | ✅ サポート中 |

### 脆弱性の報告

セキュリティ上の脆弱性を発見された場合は、公開のIssueではなく、
以下の方法で報告してください：

1. **推奨**: GitHubのSecurity Advisoryを使用
   - このリポジトリの「Security」タブから「Report a vulnerability」を選択
2. **代替**: 実行委員会へ直接連絡
   - メール: （後日追加予定）

### 報告時に含めていただきたい情報

- 脆弱性の種類（XSS、CSRF、情報漏洩など）
- 影響を受けるファイルやコンポーネント
- 再現手順
- 可能であれば、修正案や回避策

### 対応プロセス

1. **報告の受領**: 24時間以内に受領確認
2. **初期評価**: 3営業日以内に影響範囲を評価
3. **修正対応**: 重大度に応じて1-7日以内に対応
4. **公開**: 修正完了後、適切なタイミングで公開

### セキュリティのベストプラクティス

本プロジェクトで実装しているセキュリティ対策：

- ✅ 依存関係の自動更新（Dependabot）
- ✅ セキュリティヘッダーの設定（CSP、X-Frame-Options等）
- ✅ HTTPS強制（GitHub Pages）
- ✅ `.gitignore`による機密情報の除外
- ✅ 定期的な`npm audit`実行
- ✅ TypeScript strict mode
- ✅ ESLint設定

### 除外されるリスク

本プロジェクトは静的サイトのため、以下は該当しません：

- バックエンドAPI
- データベース
- ユーザー認証・認可
- 決済処理

### 連絡先

セキュリティに関する一般的な質問：

- GitHub Issues（公開情報のみ）
- 実行委員会（メールアドレスは後日追加予定）
